# คู่มือการเตรียมความพร้อมก่อนนำเสนอผลงาน (Setup & Presentation Guide)

เอกสารนี้จะช่วยให้คุณเตรียมเครื่องคอมพิวเตอร์ให้พร้อมสำหรับการพรีเซนต์งานแบบ Local ซึ่งจะมีความรวดเร็วและเสถียรที่สุดครับ

## 1. การติดตั้งโปรแกรมที่จำเป็น (ครั้งแรกเท่านั้น)
หากเครื่องที่ใช้พรีเซนต์ยังไม่มีโปรแกรมเหล่านี้ ให้ติดตั้งให้เรียบร้อยครับ:
1. **Node.js**: (แนะนำเวอร์ชัน 18 หรือ 20 LTS) ดาวน์โหลดได้ที่ [nodejs.org](https://nodejs.org/)
2. **VS Code**: สำหรับการรันคำสั่งและเปิดดูโค้ด

---

## 2. ขั้นตอนการเตรียมระบบให้พร้อมพรีเซนต์
เปิด Terminal ใน VS Code (กด `Ctrl + ~`) และทำตามลำดับดังนี้:

### ขั้นที่ 1: ติดตั้ง Library ของระบบ
คำสั่งนี้จะทำการติดตั้งไฟล์งานที่จำเป็นทั้งหมดลงในเครื่อง
```bash
npm install
```

### ขั้นที่ 2: เตรียมฐานข้อมูลและบัญชีทดสอบ
เพื่อให้มั่นใจว่าข้อมูลโครงการและรายชื่อผู้ใช้งาน (เช่น `student@example.com`) พร้อมใช้งาน ให้รันคำสั่งนี้ครับ:
```bash
npx prisma db push
npx prisma db seed
```
*หมายเหตุ: หากใช้ Windows แล้วเจอข้อความสีแดงเรื่อง Security ให้ใช้คำสั่งนี้แทน:*
```bash
node --loader ts-node/esm prisma/seed.ts
```

### ขั้นที่ 3: เริ่มรันระบบ
เมื่อเตรียมข้อมูลเสร็จแล้ว ให้เริ่มรันโปรแกรมด้วยคำสั่ง:
```bash
npm run dev
```
จากนั้นเปิด Browser (Chrome/Edge) ไปที่: **[http://localhost:3000](http://localhost:3000)**

---

## 3. คำแนะนำในการพรีเซนต์
เพื่อให้การสาธิตลื่นไหลที่สุด แนะนำให้เตรียมตัวดังนี้ครับ:

1. **เปิด 2 Browser หรือ 2 หน้าต่าง (Incognito)**:
   * **หน้าต่างที่ 1**: Login เป็น "นักศึกษา" (`student@example.com`) เพื่อแสดงการสร้างโครงการ
   * **หน้าต่างที่ 2**: Login เป็น "ผู้อนุมัติ" (เช่น `advisor@example.com`) เพื่อแสดงการกดอนุมัติทันทีโดยไม่ต้อง Login เข้า-ออกบ่อยๆ
2. **เตรียมไฟล์ทดสอบ**: เตรียมไฟล์ PDF หรือรูปภาพขนาดเล็กไว้ 1-2 ไฟล์ เพื่อใช้สาธิตการแนบเอกสาร (025/027)
3. **ใช้ WORKFLOW_GUIDE**: เปิดไฟล์ `WORKFLOW_GUIDE.md` ไว้เป็นหน้าจอสำรอง เพื่อดูว่าขั้นตอนต่อไปใครต้องเป็นคนอนุมัติ

---

## 4. การแก้ไขปัญหาเบื้องต้น
* **พอร์ต 3000 ถูกใช้งานอยู่**: หากรันแล้วแจ้งว่า Port 3000 used ให้กด `Ctrl + C` เพื่อหยุดโปรแกรมเก่าก่อนแล้วรันใหม่
* **ข้อมูลไม่ขึ้น**: ให้ลองรัน `npx prisma db seed` อีกครั้งเพื่อล้างข้อมูลเก่าและเริ่มต้นใหม่ด้วยชุดข้อมูลทดสอบมาตรฐาน
